Reservation Sync

Connected to the real reservation source

NightOS is already wired to the current operational reservation site: https://cova-club-resa-production.up.railway.app/. The current live path uses Socket.IO to read week snapshots in a way that respects the reservation site as the writer.

Current useful payload

  • occupied table slots
  • supplemental booking slots
  • source label text
  • price / spend fields

Current limitations

  • no verified client identity
  • no structured CRM-grade preferences
  • no explicit RP foreign key model yet

Current sync behavior

  • maps slots to external_booking_id
  • keeps source metadata visible
  • lets operators inspect any live week window
  • can trigger live source checks from settings
  • leaves CRM empty instead of inventing clients
